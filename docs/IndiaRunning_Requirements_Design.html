Project Overview
RunBhaarat is a full-stack web platform for discovering, registering, and managing running events across India. It serves three primary user groups: Runners who browse and register for events, Organisers who list and manage events, and Administrators who moderate the platform.
🏃
Runners
Discover nearby events, filter by city or distance category, register online, download bibs & certificates, and share race photos.
🗂️
Organisers
List events, manage participant registrations, set pricing tiers, publish results, and upload event photos via a dedicated dashboard.
🛡️
Admins
Approve event listings, manage users and flagged content, view platform analytics, and configure site-wide settings.

Core Features
🔍
Event Discovery
Search & filter by city, date, distance type (5K, 10K, HM, FM, Ultra), and price range.
📅
Event Calendar
Monthly calendar view showing upcoming races with colour-coded distance categories.
🎟️
Online Registration
Multi-tier pricing, coupon codes, group registrations, and Razorpay/UPI payment gateway.
📍
City-based Filters
Geolocation to auto-detect city; filter events within configurable radius from user's location.
⭐
Ratings & Reviews
Post-event ratings (1–5 stars), written reviews, and aggregate event score display.
📸
Race Photos
Bib-number photo search using AI face/tag matching; downloadable high-res photos.
🏅
Certificates
Auto-generated personalised finisher certificates with timing, name, and event branding.
📊
Results & Timing
Import chip-timing CSV files; display live & final results with leaderboard by category.
👤
Runner Profile
Personal dashboard with participation history, PBs, medals won, and upcoming registrations.
📣
Organiser Dashboard
Event creation wizard, registration management, revenue reports, and bulk bib allocation.
🔔
Notifications
Email & SMS reminders: registration confirmation, event reminders (7 days, 1 day), results ready.
📱
Mobile-first UI
Responsive design optimised for mobile browsers; PWA capabilities for offline bib access.

Non-Functional Requirements
⚡
Performance
LCP < 2.5s on 4G mobile. API p95 response < 300ms. Event listing page supports 500 concurrent users without degradation. Images served via CDN with lazy loading.
🔒
Security
HTTPS enforced everywhere. JWT tokens expire in 7 days. Razorpay signature verification on webhooks. OWASP Top 10 mitigations. Rate limiting: 100 req/min per IP.
📈
Scalability
Horizontal scaling for API servers. Database read replicas for listing queries. Redis caching for event listings (TTL 5 min). Queue-based certificate generation (BullMQ).
♿
Accessibility
WCAG 2.1 AA compliance. Keyboard-navigable UI. ARIA labels on interactive elements. Minimum 4.5:1 colour contrast ratio. Screen-reader tested with NVDA.
🌐
SEO
Server-side rendered event pages via Next.js. Structured data (JSON-LD: Event schema). Sitemap.xml auto-generated. OG images per event for social sharing.
🛡️
Reliability
99.5% uptime SLA. Daily automated DB backups retained 30 days. Payment webhook retries with idempotency keys. Graceful degradation when Razorpay is down.
